2. 字句解析の要素

この章では、Jedaプログラミング言語での、単語認識にもちいられる字句解析 構成について説明します。

2.1 空白

ASCIIの文字で、スペース、改行、改ページ、タブは、空白文字です。これらの 文字は、隣り合うトークンを分離する役割以外には、なにもしません。

2.2 コメント

Jedaでは、２種類のコメントのタイプがサポートされています。

従来のC言語式コメントは、/* によって始まり、 */ によって終ります。

        /* This  is a comment */
        /**********************
         *  This is a multi-line comment
         ***********************/

一行コメントは、// によって始まり、改行までのすべてのテキストはコメント とみなされます。
        //  A single Line comment

2.5 識別子

識別子は、プログラムの要素の名前を表す、一連の文字列です。 識別子は、レター文字か '_'(アンダースコア)で始まり、ゼロ個以上のレター文字、 数字、あるいは'_'が続きます。識別子の正規表現は、次のようになります。

   [a-zA-Z_][a-zA-Z_0-9]*

2.5 定数

    様々なタイプの定数が、値を直接指定するために用いられます。

2.5.1 整数定数

整数定数は、ゼロでない数字ではじまりゼロ個以上の数字が続くか、単に 0 (ゼロ) です。

    [1-9][0-9]*
    0

2.5.2 C言語スタイル数値定数

C言語スタイルの数値定数は、0(ゼロ)とベースを表す文字ではじまり、数値 文字が続きます。 十進数は d または D、八進数は o または O、十六進数は x または X、二進数は b または B がベース文字としてつかわれます。

    0[dD][0-9]+             // decimal constant
    0[oO][0-7]+             // octal constant
    0[xXhH][0-9a-fA-FxXzZ]+ // hex constant
    0[bB][01xXzZ]+          // binary constant

C言語スタイルの定数は、ビット長が３２ビットであると想定しています。もし、 ３２ビット以上の桁が指定された場合、上位ビットはオーバーフローし、コンパイラ は警告メッセージを発します。
 

2.5.3 Verilogスタイル数値定数

Verilogスタイルの数値定数は、オプションのビット長、 ' (クオート)文字、ベース 文字ではじまり、そして桁をあらわす文字列が続きます。 十進数は d または D、八進数は o または O、十六進数は x 、X 、h 、または H、二進数は b または B がベース文字としてつかわれます。

[0-9]*'[dD][0-9]+             // decimal constant
[0-9]*'[oO][0-7]+             // octal constant
[0-9]*'[xXhH][0-9a-fA-FxXzZ]+ // hex constant
[0-9]*'[bB][01xXzZ]+          // binary constant

Verilogスタイルの定数は、指定された長さのビット長となります。もし、ビット長 よりも長い桁のデータが指定されたばあい、オバーフローとなり、コンパイラは 忠告メッセージを発します。

 e.g. 3'hf, the msb bit overflows
２進数、１６進数の定数の最上位桁が X か Z であったばあい、この最上位桁は 与えられたビット長をうめるだけ拡張されます。
e.g.
32'bx は 32'bxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx と等しい。
32'hZ は 32hZZZZZZZZ と等しい。

2.5.4 浮動小数点定数

浮動小数点定数は、小数点、符号つき指数、あるいは両方をもちいて 表現されます。

    [0-9]+.[0-9]*{[eE][+-]?[0-9]+}?
    .[0-9]+{[eE][+-]?[0-9]+}?
 

e.g. 1234.56E-3

2.5.5 文字列定数

文字列定数は、" (ダブルクオート)にかこまれた文字の集まりで表現されます。

     " {character}* "

e.g."foo bar";
複数行にわたる文字列定数は、中間行の最後を \ (バックスラッシュ)で終らせる ことで指定できます。
e.g. "foo \
      bar ";
