2. Lexical Elements

This section specifies the lexical structure of the Jeda programming language.

2.1 White space

The  ASCII space,  form feed(newline) and tab  define 'white space' characters. These characters are ignored except the role as a separator of adjacent tokens.

2.2 Comments

Jeda supports two types of comment structure

A traditional C type comment starts with  /*  and ends with  */ .

        /* This  is a comment */
        /**********************
         *  This is a multi-line comment
         ***********************/

A single line comment  which starts with //  and all text until the end of line is ignored.
        //  A single Line comment

2.5 Identifiers

An identifier is a sequence of characters that represents a name of the program element. An identifier must start with  either  a letter  or the ASCII  character  '_' (underscore) and followed by zero or more letters, digits, or '_'. The regular expression of an identifier is given as:

   [a-zA-Z_][a-zA-Z_0-9]*

2.5 Constant

    Various type of constants can be used to specify a value directly.

2.5.1 Integer Constant

Integer constant starts with non zero digit followed by any digit, or just zero.

    [1-9][0-9]*
    0

2.5.2 C Style Numerical Constant

C style numerical constant starts with 0 and a base character followed by digit tokens. The decimal (d or D), octal (o or O), hex ( h or H or x or X ), and binary ( b or B ) base characters can be used to represent a constant in different bases.

    0[dD][0-9]+             // decimal constant
    0[oO][0-7]+             // octal constant
    0[xXhH][0-9a-fA-FxXzZ]+ // hex constant
    0[bB][01xXzZ]+          // binary constant

C style constant assumes that the bit length of the result are 32 bit. If more than 32 bit worth of digit are specified, upper digit will be overflowed, and compiler will report it as a warning.
 

2.5.3 Verilog Style Numerical Constant

Verilog style numerical constant starts with the optional length as the number of bits for the constant, followed by  ' and a base character, then the number characters. The decimal ('d or 'D), octal ('o or 'O), hex ( 'h or 'H or 'x or 'X ), and binary ( 'b or 'B ) base characters can be used to represent a constant in different base.

[0-9]*'[dD][0-9]+             // decimal constant
[0-9]*'[oO][0-7]+             // octal constant
[0-9]*'[xXhH][0-9a-fA-FxXzZ]+ // hex constant
[0-9]*'[bB][01xXzZ]+          // binary constant

Verilog style constant generates the specified length of the number in bits. If a digit takes more than the specified length then upper digit will overflow and compiler report it as a warning.

 e.g. 3'hf, the msb bit overflows
When binary or hex constant is declared with most significant  digit with X or Z, then the most significant digit is expanded to fill the given size.
e.g.
32'bx is equivalent to 32'bxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
32'hZ is equivalent to 32hZZZZZZZZ.

2.5.4 Floating point Constant

Floating point constant can be written with a decimal point, a signed exponent, or both.

    [0-9]+.[0-9]*{[eE][+-]?[0-9]+}?
    .[0-9]+{[eE][+-]?[0-9]+}?
 

e.g. 1234.56E-3

2.5.5 String Constant

A string constant is a sequence of characters enclosed in double quotes.

     " {character}* "

e.g."foo bar";
Multiple line string constant can be specified with the terminated with \ (backslash).
e.g. "foo \
      bar ";
